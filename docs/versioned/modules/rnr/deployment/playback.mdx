# Deploy Playback namespace

-   Navigate to manifest/playback folder
-   Fullfil and correct variables in `playback-configmap.yml`

:::info
Configration: [Config](../configuration/playback.mdx)
:::

-   Create a playback configmap

```bash
kubectl apply -f playback-configmap.yml
```

-   Create a playback deployment

```bash
kubectl apply -f playback-deployment.yml
```

-   Create a playback service

```bash
kubectl apply -f playback-service.yaml
```

-   Create an ingress

```bash
kubectl apply -f playback-ingress.yml
```


### Minimum Resource Requirements

#### MsgOp module

- The MsgOp component requires significant computational resources due to its intensive initialization and playback session processing. Resource allocation is crucial for optimal performance.
- For example: playback in speed 2x and 4x (500 entities update per sec in 10 mins)
    - Request:
        -   CPU: 1
        -   Memory: 3GiB

#### DbOp module

- Suggestion:
    - Request:
        -   CPU: 0.25
        -   Memory: 128MB

#### ReplayMgr module
- Suggestion:
    - Request:
        -   CPU: 0.125
        -   Memory: 64MB
